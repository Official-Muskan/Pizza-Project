# 🍕 Pizza Sales Analysis (SQL Workbench) 

## 📌 Project Overview 

This project focuses on analyzing pizza sales data using SQL Workbench. The dataset contains detailed information about customer orders, pizza categories, sizes, and sales performance. The goal of this project is to derive meaningful insights into sales trends, customer preferences, and revenue distribution.

The analysis was originally documented in a Word (.doc) file, and this repository serves as a structured version for sharing and collaboration.

## 📂 Dataset Information 

The dataset includes the following key attributes:

Order ID – Unique identifier for each order.

Order Date & Time – Timestamp of the order.

Pizza Name – Specific pizza ordered.

Pizza Category – Type of pizza (Classic, Supreme, Veggie, etc.).

Quantity – Number of pizzas ordered.

Total Price – Revenue generated per order. 

## 🛠️ Tools & Technologies 

SQL Workbench – Query execution and analysis.

MySQL Database – Data storage and management.

Data Documentation – Initial analysis recorded in .doc format. 

## 📊 Key Insights
Some of the queries and insights explored include:

Total revenue generated across all orders.

Best-selling pizza categories and sizes.

Peak order times and days.

Customer purchase behavior and preferences.

Contribution of each pizza type to overall sales.
